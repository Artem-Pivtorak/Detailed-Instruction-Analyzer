Звуки мають бути різними для різних типів дій (наприклад, легкий «піп» при наведенні, короткий «клік» при натисканні).

Гучність звуків регулюється перемикачем «Sound Off/On» у розділі Settings (якщо вимкнено, звуки не програються).

Консольний текст: текст над скляним фреймом у головному вікні має отримуватися з консолі (для тесту можна просто виводити випадковий рядок при запуску).

Заглушки: усі дії, що мають бути реалізовані пізніше (редагування API ключів, збереження, додавання команд, зміна мови тощо), позначаються як заглушки. Вони можуть виводити повідомлення в консоль або просто змінювати свій стан (наприклад, зміна тексту на плашці Language циклічно).

Хрестики закриття: кожен розділ (що відкривається з шестикутника) має у верхньому правому куті хрестик для повернення до головного вікна (або закриття цього розділу).

5. Технічні вимоги до реалізації
Мова програмування: C++.

Рекомендований фреймворк: Qt (з використанням QML для складних анімацій та ефектів скла) або будь-який інший, що дозволяє створити гнучкий інтерфейс із власними анімаціями (наприклад, OpenGL-контекст).

Платформа: крос-платформеність (Windows, macOS, Linux) бажана, але не обов'язкова.

Графічні ефекти: скло (blur, transparency), світіння (glow), анімації частинок (для сфер у Settings) мають бути реалізовані з використанням сучасних графічних можливостей (OpenGL / DirectX / Vulkan через Qt Quick 3D або аналог).

Звуки: відтворення через відповідну бібліотеку (наприклад, OpenAL, FMOD, або Qt Multimedia). Файли звуків надаються окремо або мають бути підібрані розробником (стиль: техно, футуристичні).

6. Примітки щодо наданих зображень
Усі розташування елементів (шестикутники, фрейми, іконки) мають точно відповідати композиції на наданих фотографіях. Розробник має отримати доступ до цих зображень (фото 1–6) для точного копіювання дизайну.

Кінцева мета: отримати повністю функціональний прототип інтерфейсу, який у майбутньому буде інтегровано з реальною логікою програми. Зараз важливо реалізувати всі візуальні ефекти, анімації та інтерактивність із заглушками замість реальних дій.
